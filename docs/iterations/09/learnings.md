# Learnings 09 — SSE и Node-стримы

## Решения и почему

- **Отдельный метод `stream()` в узком интерфейсе, а не `create({stream: true})`
  (2026-07-23).** У SDK есть оба пути, но оверлоад `create` (возврат зависит от флага в
  параметрах) в узком структурном интерфейсе заставил бы фейк кастовать — вся прелесть
  «фейк — объект-литерал без каста» умерла бы. Отдельный метод даёт два простых
  сигнатурных контракта; событие стрима описано минимальным `ResponseStreamEventLike`
  (`{type, delta?}`) — ровно то, что читает хендлер.
- **SSE-ответ — `Readable.from(async-генератор)`, мимо Zod-сериализатора.** Fastify
  пайпит Node-стрим в сокет сам; `reply.raw.write()` руками (как предполагала спека) не
  понадобился — генератор `sseFrames` выразил то же декларативнее, а обработку
  backpressure Fastify взял на себя. Это буквальный аналог
  `StreamingResponse(generator)` из FastAPI.
- **Ошибка после старта стрима — SSE-кадром `event: error` с тем же envelope.** Статус
  200 уходит с первым кадром, HTTP-код сменить уже нельзя — единственный канал для ошибки
  это сам стрим. Код `STREAM_ERROR` добавлен в контрактный словарь как единственный код
  вне HTTP-статусов; фронт парсит кадр той же формой `ErrorResponse`, что и JSON-ошибки.
  Логирование в catch — руками (`log.error`), потому что стрим течёт мимо
  `setErrorHandler` (канон iter 3 иначе потерял бы этот случай).
- **fetch-reader, а не EventSource.** EventSource умеет только GET и автоматически
  реконнектится — для платного POST-эндпоинта reconnect означал бы повторную генерацию
  за деньги без ведома пользователя. Ручной reader поверх `fetch` — это ~30 строк с
  буфером недокачанного кадра; done без `[DONE]` трактуется как обрыв, а не как успех.
- **Общая инфраструктура ошибок вынесена, а не скопирована.** Из `customFetch` выделены
  `API_PREFIX` (единственная точка знания о срезаемом прокси префиксе) и
  `errorFromResponse` (не-ok-ответ → `ApiError` из envelope) — ручной хук использует их
  же, поэтому 404 из suggest-reply выглядит для UI ровно как 404 из Orval-хука.
- **Regex-локация nginx + `rewrite` для среза префикса.** В общей `/api/`-локации префикс
  срезается trailing slash'ем в `proxy_pass`, но regex-location URI в `proxy_pass` не
  допускает — nginx падает на старте. Канонический обход — `rewrite ^/api(/.*)$ $1 break`
  перед `proxy_pass` без URI.

## Грабли (где Node повёл себя не как Python)

- **`inject()` дружит с конечным стримом бесплатно.** Ожидался спецтанец вокруг
  тестирования стрима (в pytest пришлось бы возиться с `httpx.stream`), но фейковый стрим
  конечен — `app.inject()` просто собирает весь SSE-ответ в `response.body`, и тест
  ассертит кадры побайтно. Тестировать *инкрементальность* прихода в vitest не нужно —
  её доказывает `curl -N` в demo (правило 8).
- **Чанк сети ≠ SSE-кадр.** Соблазн парсить каждый чанк reader'а как целый кадр локально
  «работает» (кадры маленькие, приходят целиком) и ломается на любой фрагментации TCP.
  Обязательный паттерн: копить буфер, резать по `\n\n`, хвост оставлять до следующего
  чанка — тот же приём, что ручной разбор строк у `requests.iter_content`.

## Сознательные срезы и отложенные TODO

- **Черновик не персистится** — домен заморожен (правило 3): уход с тикета теряет текст,
  повторное нажатие — повторный вызов OpenAI. Для фикстуры честнее, чем таскать кэш.
- **Отмены стрима нет** — уход со страницы посреди генерации не абортит fetch;
  `# dw-lite: без AbortController → cancel по unmount` — не делаем (вне scope спеки).
- **Reconnect-логики нет** — обрыв показывается ошибкой, пользователь жмёт кнопку снова.
  Осознанно: авто-retry платного вызова — то, от чего мы ушли, отказавшись от EventSource.
- **SSE не описан в OpenAPI** — заметка №6 ROADMAP: это граница контракт-подхода,
  зафиксирована комментарием в хуке и схемой в explainer, апгрейда не планируется.
