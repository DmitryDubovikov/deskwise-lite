# Демо-смок — Итерация 00 (каркас)

**Что весь прогон доказывает и зачем.** Iter 0 не отдаёт продуктовой пользы — она кладёт
рантаймовый фундамент. Значит доказать нужно ровно одно: **фундамент реально работает, а не
только описан**. Прогон проверяет три вещи, на которых будет стоять всё дальнейшее: (1)
`api/` — самостоятельный воспроизводимый pnpm-пакет; (2) Fastify-сервер поднимается одной
командой и отвечает по HTTP; (3) обвязка качества (Biome + tsc + vitest) зелёная и тест
гоняет роут без сети. Если это воспроизводится — iter 1 есть на чём строить schema-first API.

Все команды копипаст-исполнимы как есть из корня репозитория
`/Users/dd/projects/pet/deskwise-lite`. Стек этой итерации — просто локальный Node
(compose/БД появятся позже), поднимать нечего кроме dev-сервера в шаге 2.

---

## Шаг 1 — Зависимости ставятся идемпотентно

**Чем доказываем / зачем гоняем:** пакет должен быть воспроизводимым — повторная установка
не должна дрейфить лок-файл. Иначе «работает у меня» не переносится на CI и контейнер.

```bash
cd /Users/dd/projects/pet/deskwise-lite/api && pnpm install
```

**Ожидаемо:** вывод `Already up to date` (при первой установке — список поставленного, но
повторный прогон обязан быть no-op). После — лок-файл не изменён:

```bash
cd /Users/dd/projects/pet/deskwise-lite && git status --short api/pnpm-lock.yaml
```

**Ожидаемо:** либо пусто, либо `??` (файл ещё не закоммичен) — но **не** ` M` (модификация).
Дрейф лок-файла на повторной установке = баг воспроизводимости.

---

## Шаг 2 — Живой сервер отвечает по HTTP (продуктовая поверхность)

**Чем доказываем / зачем гоняем:** это и есть проверка, что скелет *живой*. Щупаем именно
настоящий сетевой роут через `curl` (реальная поверхность), а не импортируем функцию —
чтобы поймать всё, что живёт на сетевой границе (сериализация JSON, реальный `listen`).

```bash
cd /Users/dd/projects/pet/deskwise-lite/api && pnpm dev &
sleep 4
curl -s http://localhost:3000/health
echo
pkill -f "tsx watch"
```

**Ожидаемо:** `curl` печатает ровно `{"status":"ok"}`. Сервер поднялся через tsx-watch,
роут `GET /health` вернул объект статуса. (Строку `listening on…` в логе не жди — дефолтный
Fastify-логгер молчит до iter 3; факт ответа `curl`'а важнее лога.)

## Шаг 3 — Смок-тест роута зелёный, без сети

**Чем доказываем / зачем гоняем:** тест дёргает ту же фабрику `buildServer()` через
`app.inject()` — доказываем, что роут можно проверять в памяти, без порта и сети (фундамент
будущих `inject()`-тестов всего API; CI не будет открывать сокеты).

```bash
cd /Users/dd/projects/pet/deskwise-lite/api && pnpm test
```

**Ожидаемо:** vitest печатает `Test Files 1 passed (1)` и `Tests 1 passed (1)` — тест
`GET /health returns ok status` прошёл. Слово `passed`, не `skipped`.

## Шаг 4 — Полный гейт качества зелёный

**Чем доказываем / зачем гоняем:** одна команда `make check` — это ворота, которые будут
охранять каждую следующую итерацию. Прогоняем её целиком (Biome + tsc-strict + vitest),
чтобы убедиться: линт чист, типы в строгом режиме сходятся, тесты зелёные — всё разом.

```bash
cd /Users/dd/projects/pet/deskwise-lite && make check
```

**Ожидаемо:** последовательно — `biome check .` → `Checked N files … No fixes applied`;
`tsc --noEmit` → без вывода (значит типы ок); `vitest run` → `1 passed`. Ненулевой exit на
любом шаге завалил бы всю цель — здесь exit 0.

---

## Шаг 5 — Пакет самостоятельный, без workspaces

**Чем доказываем / зачем гоняем:** семейное правило 5 — `api/` независимый пакет, никаких
workspaces-of-packages. Показываем сам артефакт: у пакета есть свой лок-файл, а
`pnpm-workspace.yaml` (если есть) не объявляет членов-пакетов.

```bash
cd /Users/dd/projects/pet/deskwise-lite && ls api/pnpm-lock.yaml && \
  ( grep -q "^packages:" api/pnpm-workspace.yaml 2>/dev/null && echo "WORKSPACE MEMBERS DECLARED (bad)" || echo "no workspace members declared (ok)" )
```

**Ожидаемо:** `api/pnpm-lock.yaml` существует; вторая строка — `no workspace members
declared (ok)`. То есть `api/` — самодостаточный пакет, а не член мульти-пакетного
workspace. (Подробнее про присутствие `pnpm-workspace.yaml` см. `learnings.md`.)
