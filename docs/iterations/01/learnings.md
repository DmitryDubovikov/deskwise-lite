# Learnings 01 — Schema-first: Zod + type provider + OpenAPI

## Решения и почему

- **`buildServer()` стал `async` из-за порядка регистрации swagger.** `@fastify/swagger`
  собирает спеку через onRoute-хук — он должен встать **до** объявления роутов, иначе
  роуты в спеку молча не попадут. Поэтому `await app.register(fastifySwagger, ...)` стоит
  первым, а фабрика возвращает промис (и все вызовы — `src/index.ts`, тесты, emit-скрипт —
  делают `await buildServer()`). В FastAPI такой проблемы нет: там спека собирается лениво
  из роутера при первом запросе `/openapi.json`, порядок не важен.

- **`CreateTicketSchema` выводится из `TicketSchema` через `pick`/`extend`, а не пишется
  отдельно.** Одна правка поля в базовой схеме растекается на обе — меньше шансов, что
  DTO создания и DTO ответа разъедутся. Питоний аналог — Pydantic-модель с наследованием,
  только композиция здесь идёт методами схемы.

- **Дефолт `priority: "normal"` живёт в схеме, а не в хендлере.** Его подставляет
  валидация (`.default()` у Zod ≈ значение по умолчанию у поля Pydantic), поэтому он же
  автоматически попадает в OpenAPI как `default` — хендлер про приоритет вообще не знает.

- **Emit-скрипт пишет файл с финальным `\n` и закрывает app.** Перевод строки — чтобы
  повторный прогон давал байт-в-байт тот же файл (пустой `git diff` — фундамент CI-гейта
  iter 7); `app.close()` — чтобы процесс завершался сам, без слушающего сервера и Ctrl-C.

- **`openapi.json` исключён из Biome** (`api/biome.json`: `"includes": ["**", "!openapi.json"]`).
  Сгенерённый файл форматировать нельзя: его формат определяет генератор
  (`JSON.stringify(..., null, 2)`), и любое «причёсывание» сломало бы идемпотентность эмита.

- **`scripts/` добавлен в `tsconfig.json` `include`.** Иначе `pnpm typecheck` молча не
  проверял бы emit-скрипт — а он такой же код проекта, как и `src/`.

## Грабли («где Node повёл себя не как Python»)

- **Порядок плагинов имеет значение.** Главная встреченная разница с FastAPI: там
  `include_router` можно звать в любом порядке, здесь swagger-плагин обязан быть
  зарегистрирован раньше роутов, и ошибка проявляется не исключением, а **пустой спекой**.
  Лечится `await` на регистрации и комментарием в `server.ts:23-24`, объясняющим, почему
  `await` обязателен.

- **`components.schemas` в спеке пустой — схемы инлайнятся.** `jsonSchemaTransform`
  разворачивает каждую Zod-схему прямо в описание роута, а не выносит в `$ref` на
  `components`. Для наших потребителей (Orval, CI-diff) это работает; если захочется
  переиспользуемых named-схем в спеке, у `fastify-type-provider-zod` есть
  `jsonSchemaTransformObject` + реестр схем — не тащим, пока не понадобится.

- **Zod v4: `z.uuid()` — топ-уровневая функция.** В большинстве примеров в сети (и в
  памяти LLM) — `z.string().uuid()` из v3; в v4 строковые форматы подняты на верхний
  уровень (`z.uuid()`, `z.email()`), старый путь — deprecated. Сверялись с доками v4.

## Сознательные срезы (`dw-lite:`-маркеры в коде)

- `server.ts:36` — **in-memory `Map` вместо БД**: хранилище живёт до рестарта процесса.
  `# dw-lite: in-memory Map → Prisma/Postgres (iter 4)`.
- `server.ts:64` — **список — голый массив**, без `{items, total, page, limit}`.
  `# dw-lite: голый массив → {items, total, page, limit} (контракт №5, iter 5)`.
- `server.ts:82` — **404 — плоский `{message}`**, не envelope.
  `# dw-lite: plain message → error envelope №2 (iter 3)`; то же касается дефолтного
  Fastify-формата 400 — единый envelope и `setErrorHandler` придут в iter 3.
- **Swagger-UI не подключён** — спека нужна файлом для кодогена, не страницей для людей
  (решение спеки итерации, вне scope).
