# deskwise-lite — ROADMAP

> 🎯 **Цель проекта:** Node.js/TypeScript-стек в резюме минимальными затратами. Ново само
> оборудование → правило семьи инвертировано: **одна незнакомая концепция-кластер на
> итерацию**, знакомое (docker/nginx/compose/Postgres/Actions) — бесплатно. Existence-gate,
> не accuracy.

Источник истины по **порядку и составу**. Архитектурные правила — в `CLAUDE.md`. Тонкая спека
каждой итерации создаётся `/iterationStart N` в `specs/NN/`.

## Домен / фикстура

Вымышленная компания **Fernwood Supplies** (интернет-магазин канцтоваров), тикеты
клиентской поддержки (English product, Russian docs). Artefакты:
- **Модель Ticket:** `subject`, `body`, `status` (enum), `priority` (enum `low|normal|high`).
  Домен заморожен после iter 5 (CLAUDE.md, правило 3).
- **Seed ~30 синтетических тикетов** (English) — скриптом в iter 5; для AI-итераций тикеты с
  осмысленным телом (есть что суммаризовать).
- **AI-фичи (iter 8–9):** `summarize` (JSON через типизированный контракт) и
  `suggest-reply` (SSE-стриминг) — генеративные, не классификация.

## Сквозные контракты (решены 2026-07-12, до iter 0)

Потребляются несколькими итерациями — фиксируются здесь. Внутренности одиночных итераций
(тексты промптов, вёрстка UI, панели) — скоуп тонких спек.

1. **State machine статусов** (iter 5/6/8): `open → in_progress → resolved → closed` +
   reopen `resolved → in_progress`. Терминал — `closed`. Переход — только через
   `POST /tickets/:id/transition {to}`; недопустимый переход → **409** единого формата
   ошибок (№2). Матрица переходов — чистая domain-функция (правило 6), PATCH статуса в
   обход неё нет.
2. **Формат ошибок** (iter 3+, все эндпоинты): единый envelope
   `{"error": {"code": "<MACHINE_CODE>", "message": "..."}}`, схема ошибки — Zod-схема в
   OpenAPI на каждый код ответа (400/404/409/500). `# dw-lite: envelope → RFC 9457
   problem+json` — возможный апгрейд, не делаем.
3. **`openapi.json` — коммитится** в `api/openapi.json` (iter 1/6/7): генерится **только**
   скриптом `openapi:emit` (сборка приложения без слушающего сервера → сериализация спеки);
   Orval в `web/` читает этот файл (не URL); CI (iter 7) регенерит спеку и клиент и требует
   пустой `git diff`. Руками файл не правится никогда.
4. **Env-конвенция** (все итерации): префикс `DW_`, чтение только в config-модуле
   (`DW_DATABASE_URL`, `DW_OPENAI_API_KEY`, `DW_OPENAI_MODEL` — пиннёный снапшот,
   `DW_PORT`). Секреты — `.env` вне git, `.env.example` — в git.
5. **Пагинация и фильтры** (iter 5/6): offset-стиль `?page=&limit=` + `?status=`; ответ
   списка — `{items, total, page, limit}`. `# dw-lite: offset → cursor` — не делаем.
6. **Маршрутизация compose** (iter 4/6/9): nginx :8080 — `/` статика `web/dist`, `/api/` →
   `api:3000`; SSE-локация с отключённой буферизацией (`proxy_buffering off`) — требование
   iter 9 к конфигу nginx из iter 6.
7. **Детерминизм и стоимость AI** (iter 8/9): `temperature=0`, снапшот пиннится; в тестах и
   CI OpenAI всегда мок (фейковый клиент через `buildApp`), сеть в CI запрещена; live —
   копейки, батч-прогоны с расходом — спросить.

## Заметки

- **№1 — Fastify, не NestJS (2026-07-12).** NestJS — «Django мира Node» (DI-контейнер,
  декораторы, модули) и частотнее в вакансиях, но учили бы фреймворк, а не Node; Fastify —
  аналог FastAPI, и его схема-на-роуте кормит герой-пайплайн напрямую. Express отпал: без
  схем автоген OpenAPI мёртв. NestJS умеет работать поверх Fastify — путь не закрыт.
- **№2 — Zod, не TypeBox (2026-07-12).** TypeBox «роднее» (настоящий JSON Schema, ноль
  конверсий, быстрее), но резюме-частотность Zod несравнима, а Zod v4 + 
  `fastify-type-provider-zod` (поддержка v4, `jsonSchemaTransform` для @fastify/swagger)
  сводят конверсию к строке сетапа. Проверено по докам 2026-07-12.
- **№3 — pnpm workspaces выкинуты совсем (2026-07-12).** Главный плюс workspaces — общий
  пакет между api и web — у нас **вреден по дизайну**: контракт течёт через
  `openapi.json → Orval`, shared-типы превратили бы героя-пайплайн в декорацию. Остальные
  плюсы на двух пакетах копеечные, а Docker-сборка с workspaces заметно противнее
  (корневой lockfile, `pnpm deploy`). Итог: независимые пакеты + Makefile (паттерн tutors);
  из резюме уходит слово «monorepo» — приемлемо для «запаски».
- **№4 — auth вне скоупа (2026-07-12).** JWT/роли дали бы `securitySchemes` в OpenAPI и
  +1 слово, но ценой 1–2 итераций — для «запаски» не окупается. Следствие: все эндпоинты
  публичные, в showcase-README честная пометка.
- **№5 — Postgres везде, SQLite нигде (2026-07-12).** Prisma не поддерживает enum на
  SQLite, а у нас enum'ы в ядре домена (status/priority); два диалекта = тесты врут.
  GH Actions — штатный postgres service container.
- **№6 — SSE-эндпоинт вне автогена (2026-07-12).** OpenAPI/Orval не описывают стриминг —
  клиент suggest-reply пишется руками (маленький хук поверх fetch-stream/EventSource).
  Фиксируем как осознанное исключение и материал доки: «где граница контракт-подхода».
- **№7 — AI-фичи генеративные, не классификация (2026-07-12).** Классификация тикета —
  буквальная рифма с доменом triagewise; summarize/suggest-reply — нет. AI — фича фикстуры
  (CLAUDE.md, правило 4): без LangChain/evals/реестров/кассет, мок на границе.
- **История нарезки (2026-07-12).** Черновик 7 итераций → 8 (выделена Fastify-архитектура)
  → 11: iter 0 похудел (workspaces ушли), «архитектура» разделена на плагины (iter 2) и
  эксплуатационную поверхность (iter 3), Prisma получила собственную итерацию (iter 4).
  Критерий остановки: одна незнакомая тема на итерацию — достигнут, дальше не дробим
  (налог церемонии на итерацию фиксированный).
